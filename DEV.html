Dev Systems Local & Virtual Security

Mandatory controls for developer workstations, virtual machines, and containerized environments.

2.0 Architecture Overview

1. LOCAL SYSTEM (DEVELOPER MACHINES)

1.1 Device Security

Mandatory controls for company-issued devices.

1.1.1 Full Disk Encryption (Mandatory)

  • BitLocker (TPM-backed)
  • FileVault 2 (Secure Enclave)
  • Recovery keys stored in Azure AD / Jamf.

1.1.2 Hardening & Updates

  • Secure Boot Enabled
  • USB/External Boot Disabled
  • Auto-updates via Intune/Jamf

1.2 Identity & Access Controls

Least Privilege (LPP)

Developers do NOT get permanent local admin.

Admin via Just-In-Time (JIT) only (LAPS/PIM).

MFA Everywhere

Laptop: Hello/TouchID GitHub: FIDO2 VPN: Duo/Azure MFA AWS: FIDO2 Key

Recommended Hardware: YubiKey 5 Series, Feitian FIDO2

Conditional Access: Only approved, compliant devices (Intune/Okta Trust) may access GitHub, AWS, Internal VMs, and Jira.

1.3 Endpoint Protection

EDR/XDR (Mandatory)

CrowdStrike Falcon Defender ATP SentinelOne

USB Control

Mass storage disabled. Encrypted drives only.

1.4 Data Hygiene

PROHIBITED LOCALLY

Production logs, Prod DB dumps, OT telemetry, Customer PII, Raw API dumps.

ALLOWED

Masked synthetic datasets, 1Password/Keeper for secrets, VSCode Secret Storage.

Browser Isolation: Use Chrome Enterprise policies to block malicious extensions.
2. VIRTUAL SYSTEMS (VMs & CONTAINERS)

2.1 System Hardening

All base images MUST follow gold standards.

CIS Benchmarks (Lvl 1/2)

Ubuntu CIS Windows Server RHEL/CentOS Docker/K8s

Tools: Lynis, OpenSCAP, AWS Inspector

Image Strategy

  • Minimal, Immutable Images
  • Built via Packer / Image Builder
  • Disable SSH/RDP/GUI

2.2 Network Isolation

  • Segmentation: Dev cannot talk to Prod. Staging cannot talk to OT.
  • Zero Trust: Verify Identity, Device Posture, and Risk Score.

2.3 Access Controls

Bastion / Session Manager No direct VM access. Use AWS SSM (No inbound ports) or Azure Bastion.
IAM & Logging No shared creds. Full audit logs (CloudTrail/Log Analytics) for shell sessions.

2.4 Ephemeral Infrastructure

INFRASTRUCTURE AS CODE
Terraform Bicep Ansible
CONTAINERS

Minimal base images (distroless/alpine) with SBOM required (Syft, Trivy).

2.5 Secure Testing

  • DAST Tools Burp Suite, OWASP ZAP
  • Malware Sandbox Isolated VM / Firecracker
  • Secret Mgmt Vault, SSM, Key Vault
