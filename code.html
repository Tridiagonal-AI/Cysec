Code Security GitHub & DevSecOps

Controls for source code protection, secure CI/CD pipelines, and access governance.

Branch Protection & Workflow

Repository Governance

  • Pull Request Required: No direct commits to main.
  • Linear History: No force-pushes or history rewriting.
  • Approvals: Minimum 2 approvals (1 senior).
  • Code Owners: Critical modules require owner sign-off.
Commit Integrity

Enforce GPG Signed Commits to verify author identity. Block large binaries.

Secure CI/CD Pipeline

Automated Verification

SAST SCA Container Scan IaC Scan

Supply Chain Security: All dependencies pinned with version locks. Dependabot alerts enabled.

Access Control & Secrets

Secrets
  • Secret Scanning: Auto-block commits.
  • No Storage: Use Vault/Actions Secrets.
Auth
  • MFA/2FA: Mandatory.
  • SSO: Azure AD/Okta.
  • RBAC: Least privilege.
